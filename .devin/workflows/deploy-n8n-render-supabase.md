# Deploy Web VPN ke n8n + Render + Supabase (Gratis)

## Prerequisites
- Akun GitHub (gratis)
- Akun n8n (gratis tier)
- Akun Render (gratis tier)
- Akun Supabase (gratis tier)
- Kode aplikasi Web VPN di repository GitHub

## Langkah 1: Setup Supabase

### 1.1 Buat Project Supabase
1. Login ke https://supabase.com
2. Klik "New Project"
3. Isi detail:
   - **Name**: web-vpn-db
   - **Database Password**: Buat password yang kuat dan simpan
   - **Region**: Pilih region terdekat (misal: Singapore)
4. Klik "Create new project"
5. Tunggu 2-3 menit hingga project siap

### 1.2 Setup Database
1. Di dashboard Supabase, masuk ke **SQL Editor**
2. Jalankan query berikut untuk membuat tabel yang dibutuhkan:

```sql
-- Tabel users
CREATE TABLE users (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    username VARCHAR(255) UNIQUE NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Tabel vpn_sessions
CREATE TABLE vpn_sessions (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id BIGINT REFERENCES users(id),
    session_token VARCHAR(255) UNIQUE NOT NULL,
    ip_address VARCHAR(45),
    status VARCHAR(50) DEFAULT 'active',
    started_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    ended_at TIMESTAMP WITH TIME ZONE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Tabel access_logs
CREATE TABLE access_logs (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id BIGINT REFERENCES users(id),
    action VARCHAR(100),
    ip_address VARCHAR(45),
    user_agent TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Index untuk performa
CREATE INDEX idx_vpn_sessions_user_id ON vpn_sessions(user_id);
CREATE INDEX idx_access_logs_user_id ON access_logs(user_id);
CREATE INDEX idx_access_logs_created_at ON access_logs(created_at);
```

### 1.3 Dapatkan Kredensial Supabase
1. Di dashboard Supabase, masuk ke **Settings** > **API**
2. Catat informasi berikut:
   - **Project URL**: https://xxxxx.supabase.co
   - **anon public key**: kunci public
   - **service_role key**: kunci admin (jaga kerahasiaan)
3. Masuk ke **Database** > **Connection String**
4. Catat **Connection string** untuk PostgreSQL:
   ```
   postgresql://postgres:[PASSWORD]@db.xxxxx.supabase.co:5432/postgres
   ```

## Langkah 2: Setup n8n

### 2.1 Buat Akun n8n Cloud
1. Login ke https://n8n.io
2. Sign up untuk akun gratis (Starter tier)
3. Verifikasi email

### 2.2 Setup SendGrid untuk Email
1. Daftar di https://sendgrid.com (pilih Free Tier)
2. Verifikasi email dan login ke dashboard
3. Masuk ke **Settings** > **API Keys**
4. Klik "Create API Key":
   - **Name**: n8n-web-vpn
   - **Permissions**: Mail Send > Full Access
5. Copy API Key yang dihasilkan
6. Masuk ke **Settings** > **Sender Authentication**
7. Verify sender email (domain atau single sender)

### 2.3 Buat Workflow n8n
1. Setelah login, klik "New Workflow"
2. Buat workflow untuk automasi berikut:

#### Workflow: User Registration Automation
- **Trigger**: Webhook (POST /register)
  - Production URL: https://gansramadev.app.n8n.cloud/webhook/register
  - Response mode: Respond to Webhook node
  - Body fields: email (required), password (required), name (optional)

- **Node 1**: IF/Code (validasi data)
  ```javascript
  // Validasi email dan password
  const email = $input.item.json.body.email;
  const password = $input.item.json.body.password;
  const name = $input.item.json.body.name || '';

  if (!email || !password) {
    return { valid: false, error: 'Email dan password wajib diisi' };
  }

  const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  if (!emailRegex.test(email)) {
    return { valid: false, error: 'Format email tidak valid' };
  }

  return { valid: true, email, password, name };
  ```

- **Branch IF (Validation FAILS)**:
  - **Node 2a**: Respond to Webhook
    - HTTP Status: 400
    - Response Body: `{"success": false, "error": "..."}`

- **Branch IF (Validation PASSES)**:
  - **Node 2b**: Set (preserve email dan name sebelum Supabase)
    - Set fields: `email`, `name` dari webhook body

  - **Node 3**: Supabase (insert user ke tabel 'users')
    - Operation: Create row
    - Table: users
    - Columns mapping:
      - email: `{{ $json.email }}`
      - password_hash: `{{ $json.password }}` (catatan: idealnya hash password dulu)
      - name: `{{ $json.name }}` (jika ada)
    - Catatan: Password disimpan plaintext untuk sementara. Untuk production, tambahkan node hashing sebelum insert.
    - Error handling: Jika insert gagal, workflow akan stop dan error akan di-return ke webhook

  - **Node 4**: IF (cek apakah Supabase insert berhasil)
    - Condition: Jika insert berhasil (ada ID yang di-return)

  - **Branch IF (Insert FAILS)**:
    - **Node 5a**: Respond to Webhook
      - HTTP Status: 500
      - Response Body: `{"success": false, "error": "Database error: failed to create user"}`

  - **Branch IF (Insert SUCCEEDS)**:
    - **Node 5b**: SendGrid (kirim konfirmasi registrasi)
    - To: `{{ $json.email }}` (dari Set node)
    - From: verified sender email
    - Subject: Welcome to Web VPN
    - Content: HTML template dengan greeting menggunakan name

    - **Node 6**: Respond to Webhook
      - HTTP Status: 200
      - Response Body: `{"success": true, "userId": {{ $json.id }}}`

#### Workflow: VPN Session Management
- **Trigger**: Webhook (POST /start-session)
  - Production URL: https://gansramadev.app.n8n.cloud/webhook/start-session
  - Response mode: Respond to Webhook node
  - Body fields: user_id (optional), email (optional) - minimal satu harus ada

- **Node 1**: IF/Code (validasi identifier)
  ```javascript
  // Validasi user identifier
  const userId = $input.item.json.body.user_id;
  const email = $input.item.json.body.email;

  if (!userId && !email) {
    return { valid: false, error: 'User ID atau Email wajib diisi' };
  }

  return { valid: true, userId, email };
  ```

- **Branch IF (Validation FAILS - missing identifier)**:
  - **Node 2a**: Respond to Webhook
    - HTTP Status: 400
    - Response Body: `{"success": false, "error": "User ID atau Email wajib diisi"}`

- **Branch IF (Validation PASSES)**:
  - **Node 2b**: Supabase (query user dari tabel 'users')
    - Operation: Select/Read rows
    - Table: users
    - Filter: user_id atau email (berdasarkan yang tersedia)
    - Catatan: Supabase node akan replace item JSON dengan hasil query

  - **Node 3**: IF (cek apakah user ditemukan)
    - Condition: Jika hasil query kosong (no user found)

  - **Branch IF (User NOT FOUND)**:
    - **Node 4a**: Respond to Webhook
      - HTTP Status: 404
      - Response Body: `{"success": false, "error": "User not found"}`

  - **Branch IF (User FOUND)**:
    - **Node 4b**: Set (preserve user_id dari hasil query)
      - Set field: `userId` dari Supabase query result

    - **Node 5**: Code (generate JWT session token dengan Node crypto)
      ```javascript
      // Generate JWT menggunakan Node crypto module
      const crypto = require('crypto');

      // Ambil user_id dari Set node
      const userId = $input.item.json.userId;
      const jwtSecret = $env.JWT_SECRET; // Dari environment variable n8n

      // Header
      const header = {
        alg: 'HS256',
        typ: 'JWT'
      };
      const encodedHeader = Buffer.from(JSON.stringify(header))
        .toString('base64url');

      // Payload
      const now = Math.floor(Date.now() / 1000);
      const payload = {
        userId: userId,
        iat: now,
        exp: now + 3600 // 1 hour expiry
      };
      const encodedPayload = Buffer.from(JSON.stringify(payload))
        .toString('base64url');

      // Signature
      const signatureInput = `${encodedHeader}.${encodedPayload}`;
      const signature = crypto
        .createHmac('sha256', jwtSecret)
        .update(signatureInput)
        .digest('base64url');

      const jwtToken = `${signatureInput}.${signature}`;

      return {
        jwtToken: jwtToken,
        userId: userId,
        expiresAt: new Date((now + 3600) * 1000).toISOString()
      };
      ```

    - **Node 6**: Supabase (insert ke tabel 'vpn_sessions')
      - Operation: Create row
      - Table: vpn_sessions
      - Columns mapping:
        - user_id: `{{ $json.userId }}`
        - session_token: `{{ $json.jwtToken }}`
        - started_at: `{{ $now }}`
        - expires_at: `{{ $json.expiresAt }}`
        - status: 'active'

    - **Node 7**: Respond to Webhook
      - HTTP Status: 200
      - Response Body: `{"success": true, "session": {"token": "{{ $json.jwtToken }}", "userId": {{ $json.userId }}, "expiresAt": "{{ $json.expiresAt }}"}}`

### 2.4 Dapatkan Webhook URLs
1. Untuk setiap workflow, aktifkan webhook
2. Catat webhook URL yang dihasilkan:
   - Registration: https://n8n-instance.webhook.site/register
   - Session: https://n8n-instance.webhook.site/start-session

### 2.5 Setup Environment Variables di n8n
1. Masuk ke **Settings** > **Environment Variables**
2. Tambahkan:
   - `SUPABASE_URL`: [Supabase Project URL]
   - `SUPABASE_KEY`: [Supabase service_role key]
   - `JWT_SECRET`: [generate random secret yang kuat, misal dengan `openssl rand -base64 32`]
   - `SENDGRID_API_KEY`: [API Key dari SendGrid]

## Langkah 3: Setup Render

### 3.1 Persiapkan Repository GitHub
1. Pastikan kode Web VPN sudah di GitHub
2. Buat file `.env.example` di root project:

```env
APP_NAME=Web-VPN
APP_ENV=production
APP_DEBUG=false
APP_URL=https://your-app.onrender.com

# Database (Supabase)
DB_CONNECTION=pgsql
DB_HOST=db.xxxxx.supabase.co
DB_PORT=5432
DB_DATABASE=postgres
DB_USERNAME=postgres
DB_PASSWORD=your_supabase_password

# Supabase API
SUPABASE_URL=https://xxxxx.supabase.co
SUPABASE_KEY=your_supabase_anon_key

# n8n Webhooks
N8N_REGISTER_WEBHOOK=https://n8n-instance.webhook.site/register
N8N_SESSION_WEBHOOK=https://n8n-instance.webhook.site/start-session

# Security
APP_KEY=base64:generate-random-32-char-key
```

3. Buat file `render.yaml` di root project:

```yaml
services:
  - type: web
    name: web-vpn
    env: php
    plan: free
    buildCommand: composer install --no-dev --optimize-autoloader
    startCommand: php artisan serve --host=0.0.0.0 --port=$PORT
    envVars:
      - key: APP_ENV
        value: production
      - key: APP_DEBUG
        value: false
      - key: DB_CONNECTION
        value: pgsql
      - key: DB_HOST
        sync: false
      - key: DB_PORT
        value: 5432
      - key: DB_DATABASE
        value: postgres
      - key: DB_USERNAME
        value: postgres
      - key: DB_PASSWORD
        sync: false
      - key: SUPABASE_URL
        sync: false
      - key: SUPABASE_KEY
        sync: false
      - key: N8N_REGISTER_WEBHOOK
        sync: false
      - key: N8N_SESSION_WEBHOOK
        sync: false
```

### 3.2 Deploy ke Render
1. Login ke https://render.com
2. Klik "New +" > "Web Service"
3. Connect repository GitHub:
   - Pilih repository Web VPN
   - Render akan otomatis mendeteksi `render.yaml`
4. Konfigurasi service:
   - **Name**: web-vpn
   - **Region**: Singapore (atau terdekat)
   - **Branch**: main/master
5. Setup Environment Variables:
   - Masukkan semua nilai dari file `.env.example`
   - Gunakan kredensial dari Supabase dan n8n
6. Klik "Create Web Service"
7. Tunggu proses build dan deploy (5-10 menit)

### 3.3 Verifikasi Deploy
1. Setelah deploy selesai, Render akan memberikan URL:
   - https://web-vpn.onrender.com
2. Buka URL tersebut untuk testing
3. Cek logs di dashboard Render jika ada error

## Troubleshooting

### Error: Database Connection Failed
- Pastikan DB_HOST dan DB_PASSWORD benar
- Cek apakah Supabase project sudah aktif
- Verifikasi connection string format

### Error: Webhook Timeout
- Cek apakah n8n workflow aktif
- Verifikasi webhook URL benar
- Pastikan n8n instance tidak down

### Error: Build Failed di Render
- Cek build logs di dashboard Render
- Pastikan `composer.json` valid
- Verifikasi semua dependencies terinstall

### Error: 504 Gateway Timeout
- Ini normal di tier gratis Render (cold start)
- Tunggu beberapa detik dan refresh
- Consider upgrade ke paid tier untuk production

## Limitations Gratis Tier

### Render Free Tier
- 512 MB RAM
- Shared CPU
- Sleep setelah 15 menit tidak aktif (cold start ~30 detik)
- 750 jam build time per bulan

### Supabase Free Tier
- 500 MB database storage
- 1 GB bandwidth
- 2 projects
- 50,000 API requests per bulan

### n8n Starter Tier
- 5,000 workflow executions per bulan
- 1 active workflow
- Basic integrations

## Tips Optimasi

1. **Minimize Cold Starts**: Setup cron job untuk ping setiap 10 menit
2. **Optimize Database**: Gunakan indexes di Supabase
3. **Cache Responses**: Implement caching di aplikasi
4. **Monitor Usage**: Track API calls untuk tidak exceed limit
5. **Backup Data**: Export database Supabase secara berkala
